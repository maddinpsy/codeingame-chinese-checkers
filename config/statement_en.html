The Goal

Chinese checkers is a turn-based board game. The objective is to be first to race all pieces across the board.

  Rules

The game is played on a hexagonal star board with three players.

To address the locations in the hexagonal grid cube coordinates are used. Each direction in the grid corresponds to one axis.
  • From the bottom to top: q-axis
  • From bottom left to the top right: r-axis
  • From top left to the bottom right: s-axis.
Resulting in the coordinates ( q, r, s ). The mapping from 3D coordinates to the 2D plane leads to the constraint: q + r + s = 0.

The central position C is ( 0, 0, 0 ). Each player has ten pieces and starts in one of the six corners of the star. The coordinates in the start corner are positive in the players axis. For player 1's start position, the first coordinate is positive. For player 2 the second and player 3 the third.
Player 1 starts in the bottom corner. With pieces on following coordinates:
( 3, -3, 0 ), ( 3, -2, -1 ) , ( 3, -1, -2 ), ( 3, 0, -3 )
        ( 4, -3, -1 ), ( 4, -2, -2 ) , ( 4, -1, -3 )
                ( 5, -3, -2 ), ( 5, -2, -3 )
                        ( 6, -3, -3 )

Player 2 starts in the top right corner. With pieces on following coordinates:
( -3, 3, 0 ), ( -2, 3, -1 ) , ( -1, 3, -2 ), ( 0, 3, -3 )
        ( -3, 4, -1 ), ( -2, 4, -2 ) , ( -1, 4, -3 )
                ( -3, 5, -2 ), ( -2, 5, -3 )
                        ( -3, 6, -3 )

Player 3 starts in the top left corner. With pieces on following coordinates:
( 0, -3, 3 ), ( -1, -2, 3 ) , ( -2, -1, 3 ), ( 0, -3, 3 )
        ( -1, -3, 4 ), ( -2, -2, 4 ) , ( -3, -1, 4 )
                ( -3, -2, 5 ), ( -2, -3, 5 )
                        ( -3, -3, 6 )


They try to race all pieces into the opposite corner of the star. Players take turns moving a single piece:
  • A piece can be pushed in any direction to an adjacent empty space.
  • A piece can jump over another piece, no matter the color. You can chain consecutive hops in a single move.
A player may not combine jumping with a push move - a move consists of one or the other. There is no capturing, so pieces that are hopped over remain in play.

Jumping along the red path.
Ending on (-1, 0, 1).
The output must be 3 -2 -1 -1 0 1.

Player 2 is moving the blue 2 at (-1, 3, -2).

  Game Input

There is no initialization data. All data is sent in every move.
Input

Line 1: An integer N, the number of entities. Currently always 30.

Next N lines: Four integers separated by space Id, q,r,s representing the player number and position of all pieces.

Output
A single line containing the start position and end position of the move separated by space. q_start r_start s_start q_end r_end s_end.
Constraints
N == 30
1 ≤ id ≤ 3
-6 ≤ q,r,s ≤ 6
q+r+s == 0
